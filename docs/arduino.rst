Arduino And STM32
=================

Packed library
--------------

Arduino builds use the packed library in ``libcxxcanard/src``. The modular
headers in ``libcxxcanard/cyphal`` are for normal CMake projects.

The packed library is generated by:

.. code-block:: bash

   python3 libcxxcanard/arduino_pack.py

``arduino_pack.py`` reads ``arduino_build.json``, writes ``src/cyphal.h`` and
``src/cyphal.cpp``, and copies ``libcanard`` and ``o1heap`` into ``src/``.
Generated DSDL C headers and C++ traits are expected to be present under
``src/uavcan``, ``src/reg``, ``src/voltbro``, or other namespace directories.

Includes
--------

Arduino sketches include the packed header and generated ``.hpp`` type traits:

.. code-block:: cpp

   #include <VBCoreG4_arduino_system.h>
   #include <cyphal.h>

   #include <uavcan/primitive/scalar/Natural32_1_0.hpp>

   using Natural32 = uavcan_primitive_scalar_Natural32_1_0;

Embedded integration
--------------------

``ArduinoCyphal<>`` is the STM32 Arduino integration wrapper. A sketch usually
overrides:

* ``setup_subscriptions()`` to instantiate subscriptions.
* ``in_loop_reporting()`` to publish periodic telemetry.

The main loop calls ``cyphal_loop()``:

.. code-block:: cpp

   void loop() {
       cyphal->cyphal_loop();
   }

Registers and node info
-----------------------

``ArduinoCyphal<N>`` can expose ``N`` Cyphal registers. Pass an
``std::array<RegisterDefinition, N>`` to the constructor. The built-in embedded
stack also handles standard node info and heartbeat publishing.

See ``examples/full_example.ino`` for a compact sketch with:

* Natural32 subscription and publishing.
* ``voltbro.echo.echo_msg`` publishing.
* ``voltbro.echo.echo_service`` responses.
* Cyphal register access.
